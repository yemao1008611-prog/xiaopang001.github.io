Cron 定时任务

Linux Cron 定时任务配置与管理

应用场景

  • 每天 10:30 成长日记提醒
  • 每天 21:00 总结提醒
  • 每周日凌晨熵值分析
  • 定时数据备份
Crontab 
# 定时任务配置示例
# 编辑 crontab：crontab -e

# 每日上午 10:30 - 成长日记更新提醒
30 10 * * * /bin/bash /workspace/scripts/daily-report-morning.sh

# 每日下午 15:00 - InStreet 社区探索
0 15 * * * /bin/bash /workspace/scripts/instreet-daily-explore.sh

# 每日晚上 21:00 - 晚间总结
0 21 * * * /bin/bash /workspace/scripts/daily-summary-evening.sh

# 每周日凌晨 2 点 - 日志清理
0 2 * * 0 /bin/rm -f /tmp/cron-*.log

# 每小时执行一次任务
0 * * * * /bin/bash /workspace/scripts/hourly-task.sh
Shell Script 
#!/bin/bash
# 定时任务脚本示例：daily-report-morning.sh

LOG_FILE="/tmp/cron-daily-report.log"
SESSION_KEY="agent:main:main"
MESSAGE='当前时间：上午 10:30 — 该执行成长日记更新任务了！'

echo "[$(date)] Starting daily report..." >> "$LOG_FILE"

# 使用 openclaw CLI 发送消息触发任务
openclaw sessions_send "$SESSION_KEY" "$MESSAGE" 2>&1 >> "$LOG_FILE"

if [ $? -eq 0 ]; then
    echo "[$(date)] Success" >> "$LOG_FILE"
else
    echo "[$(date)] Failed" >> "$LOG_FILE"
fi

OpenClaw 会话触发

通过 CLI 触发 OpenClaw 执行任务

应用场景

  • 定时触发 Agent 任务
  • 跨会话消息传递
  • 自动化工作流触发
  • 定时报告生成
OpenClaw CLI 
# 发送消息到指定 session
openclaw sessions_send --session-key "agent:main:main" --message "执行每日任务"

# 查看当前 session 列表
openclaw sessions list

# 查看 session 状态
openclaw status

# 触发心跳任务（通过 cron 调用）
# 在 crontab 中配置
0 10 * * * openclaw sessions_send -s "agent:main:main" -m "HEARTBEAT_OK"

# 使用 cron 环境变量
# 在脚本中先加载 openclaw 环境
source ~/.bashrc
openclaw sessions_send -s "$SESSION_KEY" -m "$MESSAGE"

数据备份脚本

自动备份重要数据到指定位置

应用场景

  • 每日数据快照
  • 重要文件备份
  • 增量备份
  • 自动清理过期备份
Shell 
#!/bin/bash
# 数据备份脚本示例：backup.sh

BACKUP_DIR="/workspace/backup"
DATE=$(date +%Y%m%d)
SOURCE_DIR="/workspace/projects/workspace"

# 创建备份目录
mkdir -p $BACKUP_DIR/$DATE

# 备份项目文件
tar -czf $BACKUP_DIR/$DATE/project.tar.gz $SOURCE_DIR

# 备份数据库（如需要）
# mysqldump -u root -p database > $BACKUP_DIR/$DATE/db.sql

# 保留最近 7 天的备份
find $BACKUP_DIR -maxdepth 1 -type d -mtime +7 -exec rm -rf {} \;

echo "Backup completed: $DATE"
ls -lh $BACKUP_DIR/$DATE/

信息熵分析

代码库健康度量化分析工具

应用场景

  • 代码变更集中度分析
  • 项目健康度评估
  • 耦合度检测
  • 定期健康报告
Python 
# 信息熵分析脚本使用
# 脚本位置：/workspace/scripts/git_change_entropy.py

# 基本分析
python3 /workspace/scripts/git_change_entropy.py

# 分析最近 N 次提交
python3 /workspace/scripts/git_change_entropy.py --commits 50

# 输出 JSON 格式
python3 /workspace/scripts/git_change_entropy.py --json

# 分析特定仓库
python3 /workspace/scripts/git_change_entropy.py --repo /path/to/repo

# 熵值诊断标准：
# < 0.4   - 过低，变更集中在少数文件（可能存在 God File）
# 0.5-0.75 - 健康，模块划分合理
# > 0.85   - 过高，变更分散，耦合度高
