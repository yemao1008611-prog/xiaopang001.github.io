浏览器自动化

通过代码控制浏览器自动执行操作

应用场景

  • 自动填写表单
  • 批量点击操作
  • 网页截图采集

OpenClaw Browser 
# 打开网页
browser action=open url="https://example.com"

# 截图
browser action=snapshot

# 点击元素（需要先获取 targetId）
browser action=act targetId=xxx kind=click

# 输入文本
browser action=act targetId=xxx kind=type text="Hello World"

# 等待元素加载
browser action=act targetId=xxx kind=wait timeMs=5000

# 滚动页面
browser action=act kind=scroll height=1000

# 关闭浏览器
browser action=close
Python (Selenium) 
# Selenium 浏览器自动化示例
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.chrome.options import Options

# 配置无头模式
options = Options()
options.add_argument('--headless')
options.add_argument('--no-sandbox')

driver = webdriver.Chrome(options=options)

# 打开网页
driver.get('https://example.com')

# 查找元素并点击
button = driver.find_element(By.ID, 'submit')
button.click()

# 获取页面内容
content = driver.page_source

# 截图
driver.save_screenshot('screenshot.png')

driver.quit()

Python (Requests) 
# 使用 Requests + BeautifulSoup 抓取
import requests
from bs4 import BeautifulSoup

url = 'https://example.com'
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
}

response = requests.get(url, headers=headers)
soup = BeautifulSoup(response.text, 'html.parser')

# 提取标题
title = soup.title.string

# 提取所有链接
links = [a['href'] for a in soup.find_all('a', href=True)]

# 提取特定元素
content = soup.find('div', class_='content').text

print(f'Title: {title}')
print(f'Links: {len(links)}')

Git Commands 
# 1. 创建 GitHub 仓库
# 访问 https://github.com/new 创建仓库

# 2. 克隆仓库到本地
git clone https://github.com/username/repo.git
cd repo

# 3. 添加网站文件
echo "Hello World" > index.html
git add .
git commit -m "Add website"

# 4. 推送到 GitHub
git push origin main

# 5. 启用 GitHub Pages
# Settings → Pages → Source: main branch → Save

# 访问：https://username.github.io/repo
自动部署 
# 本地修改后自动部署脚本
#!/bin/bash
# deploy.sh

git add .
git commit -m "Update: $(date)"
git push origin main

echo "Deployed successfully!"
# 访问 https://username.github.io/repo 查看

# 或使用 GitHub CLI
gh repo deploy

# 注意事项：
# 1. 仓库名建议使用 username.github.io
# 2. index.html 必须在根目录
# 3. 首次部署需要等待 1-2 分钟
